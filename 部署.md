# FreeTools 部署方案

## 仓库信息

- **GitHub 仓库**: `https://github.com/jasester/free-tools`
- **仓库名**: `free-tools`（普通仓库，非 `<username>.github.io`）
- **Pages URL**: `https://jasester.github.io/free-tools/`

---

## 一、部署步骤

### 第 1 步：确认代码已推送

```bash
cd tool
git status                             # 确保无未提交的文件
git log --oneline -3                   # 查看最近提交
git push origin main                   # 如果还没推送
```

### 第 2 步：配置 next.config.js 的 basePath

由于仓库名是 `free-tools`（不是 `jasester.github.io`），需要设置 `basePath` 为 `/free-tools`，否则 CSS/JS 路径会 404。

修改 `next.config.js`：

```js
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
  images: {
    unoptimized: true,
  },
  basePath: '/free-tools',      // ← 仓库名
  trailingSlash: true,
};

module.exports = nextConfig;
```

### 第 3 步：确认 GitHub Actions 工作流存在

`.github/workflows/deploy.yml` 已经创建好，确认在仓库根目录下。

### 第 4 步：在 GitHub 上启用 Pages

1. 打开 https://github.com/jasester/free-tools/settings/pages
2. **Source** 下拉选择 **GitHub Actions**
3. 点击保存

### 第 5 步：推送代码触发部署

```bash
cd tool
git add .
git commit -m "fix: update basePath for GitHub Pages deployment"
git push origin main
```

### 第 6 步：查看部署状态

1. 打开 https://github.com/jasester/free-tools/actions
2. 等待 `Deploy to GitHub Pages` 工作流运行完成（约 1~2 分钟）
3. 部署成功后访问：**https://jasester.github.io/free-tools/**

---

## 二、部署后验证

部署成功后，检查以下页面是否正常：

| 页面 | URL |
|------|-----|
| 首页 | `/free-tools/` |
| JSON 格式化 | `/free-tools/tools/json-formatter/` |
| 图片压缩 | `/free-tools/tools/image-compress/` |
| 二维码生成 | `/free-tools/tools/qr-generator/` |
| 时间戳转换 | `/free-tools/tools/timestamp/` |
| BMI 计算器 | `/free-tools/tools/bmi-calculator/` |
| 正则测试器 | `/free-tools/tools/regex-tester/` |
| Base64 | `/free-tools/tools/base64/` |

常见问题排查：

| 问题 | 原因 | 解决 |
|------|------|------|
| 页面白屏/无样式 | basePath 没设置 | 确认 `next.config.js` 中 `basePath: '/free-tools'` |
| 404 | Pages source 没选 GitHub Actions | Settings → Pages → Source 选 GitHub Actions |
| Actions 失败 | Node 版本不兼容 | deploy.yml 中 `node-version: 20` 已正确 |
| 图片没加载 | Image Optimization 在 export 模式不支持 | 已设 `images.unoptimized: true` |


## 四、自动部署说明

之后每次 `git push` 到 `main` 分支，GitHub Actions 会自动：

1. 检出代码
2. 安装依赖 (`npm ci`)
3. 构建项目 (`npm run build`)
4. 部署到 GitHub Pages

整个过程约 1~2 分钟，无需手动操作。

## 五、后续步骤

部署完成后：

1. **Google AdSense 申请** — 部署成功后可访问后去申请
2. **Google Search Console** — 添加 `jasester.github.io` 域名
3. **自定义域名（可选）** — 购买域名后在 Settings → Pages → Custom domain 中绑定
4. **替换 AdSense ID** — 搜索 `ca-pub-YOUR_PUBLISHER_ID` 替换为真实 ID，搜索 `YOUR_AD_SLOT_` 替换为广告单元 ID
